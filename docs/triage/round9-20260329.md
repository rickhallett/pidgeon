## Coder is doing now

### Fix 1: Validate numeric CLI args before calling provider

**Source:** Claude HIGH + Codex MEDIUM + Gemini HIGH (3-family consensus)
**Safe to apply now** — tightens validation, no behavior change for valid inputs

`Number("abc")` returns `NaN`, `Number("")` returns `0`, `Number(" ")` returns `0`.
These values pass through to the provider as `RateRequest.packages[0].weight.value`,
eventually serialized into the UPS JSON body. UPS rejects with an opaque error.

The CLI is the outermost service boundary. The spec says "validated at service
boundary before any external call."

**Fix:** After `Number()` coercion, guard with `Number.isFinite(n) && n > 0` for
weight and all three dimensions. Write a clear error via `deps.write` and return.

### Fix 2: Route Commander validation errors through deps.write

**Source:** Codex HIGH + Gemini LOW (2-family consensus)
**Safe to apply now** — `configureOutput.writeErr` pipes to `deps.write` instead of swallowing

Currently `configureOutput({ writeErr: () => {}, writeOut: () => {} })` silences
everything. When Commander throws for a missing required option, no message is
written — just a silent throw. The user gets no explanation.

**Fix:** Change `writeErr` to `(str) => deps.write(str)` so Commander's validation
messages (like "required option '--origin-postal <code>' not specified") surface
through the same output channel.
