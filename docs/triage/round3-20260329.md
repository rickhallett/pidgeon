### T1: Investigate and test `DeliveryTimeInformation` in request body

**Source:** Codex HIGH #1
**Priority:** High â€” directly affects whether TimeInTransit appears in response

The UPS API reference says (line 279): "TimeInTransit requires
DeliveryTimeInformation in request." Our request builder does not include this
field. Without it, UPS may return a 200 OK without TimeInTransit, which
`mapResponse` now rejects as `{ ok: false, error: "missing TimeInTransit data" }`.

**Action:**
1. Check the UPS API documentation at https://developer.ups.com/tag/Rating?loc=en_US
   for what `DeliveryTimeInformation` looks like
2. Write a test that asserts `DeliveryTimeInformation` is present in the captured
   request body
3. Coder will implement to make it green

**Source:** Codex MEDIUM #2
**Priority:** Medium â€” may cause 400 from UPS if mismatched

while the URL path segment handles the variant (`/Shoptimeintransit`). Our code
currently sends `RequestOption: "Shoptimeintransit"` in BOTH the URL path and
the JSON body. This may be wrong â€” the JSON field might need to be `"Shop"`.

**Action:**
1. Check UPS docs for whether `RequestOption` in JSON body should match the URL
   path segment or use the base value (`"Shop"`)
2. Update the existing test at request-builder.test.ts:199 if needed
3. Coder will update `buildRequestBody` to match
