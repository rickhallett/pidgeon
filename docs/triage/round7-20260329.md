## Coder is doing now

### Fix 1: Add `.finite()` and upper bounds to numeric validators

**Source:** Claude HIGH (Infinity/scientific notation via z.coerce.number())
**Safe to apply now** — tightens validation, no behavior change for valid inputs

`z.coerce.number()` calls `Number(input)`, which accepts `"Infinity"`, `"3e2"`,
and `"0x1F"`. `UPS_TIMEOUT_MS="Infinity"` creates a request that never times
out. `UPS_MAX_ATTEMPTS="3e2"` creates 300 retry attempts.

**Fix:** Add `.finite()` to reject Infinity/NaN. Add `.max()` bounds:
- `maxAttempts` <= 10
- `baseDelayMs` <= 30_000
- `timeoutMs` <= 60_000
- `maxRetryAfterSeconds` <= 300

### Fix 2: Trim whitespace on credential strings

**Source:** Claude MEDIUM (whitespace credentials pass min(1))
**Safe to apply now** — defensive, catches misconfigured env vars at startup

`" "` (single space) passes `z.string().min(1)`. It will be base64-encoded and
sent to UPS, which rejects it with an opaque auth error.

**Fix:** Change `z.string().min(1, ...)` to `z.string().trim().min(1, ...)`.

### Fix 3: Use injectable env parameter in tests instead of mutating process.env

**Source:** Claude MEDIUM (test isolation)
**Safe to apply now** — test refactor, no production code change

`loadUpsConfig()` already accepts an `env` parameter. Tests should pass the env
record directly instead of mutating `process.env` with `beforeEach`/`afterEach`.
Eliminates the global state risk and removes ~20 lines of save/restore boilerplate.
